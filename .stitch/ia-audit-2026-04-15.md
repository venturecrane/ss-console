## A. Orphan destinations

Routes that exist in `src/pages/**` but have no `<a href>` from any navigated surface that would expose them.

| Route                | Surface class       | Archetype | Problem                                                                                                                        |
| -------------------- | ------------------- | --------- | ------------------------------------------------------------------------------------------------------------------------------ |
| `/portal/quotes`     | session-auth-client | list      | Not linked from `/portal`. Only reachable via a specific quote shown in Recent Activity (detail ‚Üí back ‚Üí list ‚Äî a contortion). |
| `/portal/invoices`   | session-auth-client | list      | Same pattern. Only reachable via a specific invoice Recent Activity entry.                                                     |
| `/portal/documents`  | session-auth-client | list      | Completely orphaned. No entry point from the hub.                                                                              |
| `/portal/engagement` | session-auth-client | detail    | Completely orphaned.                                                                                                           |

**Evidence:** grep of `src/pages/portal/index.astro`:

```
‚úó href="/portal/quotes" not found
‚úó href="/portal/invoices" not found
‚úó href="/portal/documents" not found
‚úó href="/portal/engagement" not found
```

**Root cause:** portal home was authored with a hub-and-spoke _intent_ (dashboard + ActionCard + Recent Activity + ConsultantBlock) but the hub-and-spoke _required element_ ‚Äî section cards linking to every spoke ‚Äî was never implemented. v1 of the spec did not declare this requirement; v2 does (Appendix C.1.1).

**Fix:** Add a 2√ó2 section card grid to `/portal`:

| Card      | Icon         | Destination        |
| --------- | ------------ | ------------------ |
| Proposals | description  | /portal/quotes     |
| Invoices  | receipt_long | /portal/invoices   |
| Documents | folder       | /portal/documents  |
| Progress  | timeline     | /portal/engagement |

Implementation: `src/pages/portal/index.astro`, insert grid in main column between hero/headline and Recent Activity divider. Each card is a full-click `<a>` with `min-h-[88px]`, `aria-label`, icon + Plus Jakarta Sans bold label + muted caption, rounded-lg border, focus ring per Section 8 state colors.
